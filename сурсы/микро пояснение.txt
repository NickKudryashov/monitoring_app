Слои:
app - здесь глобальный стор с логикой, роутингом, стилями. высший слой
pages - здесь находятся страницы, в основном состоят из нижележащих слоев (композиция) (пример: страница прибора, объекта)
widgets - объединяют в себе entities и features (пример: дерево элементов слева)
features - пользовательские действия, используют в себе только entities (пример: кнопка опросить (нужно конкретно переработать))
entities - наши бизнесовые сущности и приблуды, которые свойственно только им (категория, объект, прибор и тд)
shared - части ui кита, инстанс апишки, хэлперы, конфиги ( используются во всех вышележащих модулях )

в каждой папке лежит как правило:
index.ts файл - паблик апи, через него отдаются внутренние элементы и скрываются те которые не должны выплывать на поверхность
папка Ui - в ней находится html код с функциями, по факту в папке лежит схема рендера компонента
папка model - в ней находится логика 
	-slice/reducer - логика и описание действий с элементом
	-service - как правило интеграция взаимодействия с api с внутренней логикой
	-types папка с описанием элемента, по факту нужна для тайпскрипта
